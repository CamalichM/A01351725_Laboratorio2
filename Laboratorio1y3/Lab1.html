Preguntas HTML

• ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es considerado como la red de dispositivos conectados entre si. La World Wide Web es el sistema de hipertextos enlazados que permite transmitir información por medio de la conectividad a internet, por lo que la segunda es codependiente de la primera.

• ¿Cuáles son las partes de un URL?

  • Protocolo: Hypertext Transfer Protocol, funciona para transferir e intercambiar información entre actores.
  • Subdominio: World Wide Web o www, principalmente fines organizativos.
  • Dominio: Nombre único que identifica al sitio web para evitar las direcciones Ip directas.
  • Ruta: Especificación del fichero, archivo o ubicación alojada en la página web.
  • Parámetros: Datos finales que permiten agregar filtros o especificaciones a la búsqueda.

• ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Recuperar datos del servidor.
  • HEAD: Envía el header sin el cuerpo.
  • POST: Enviar entidad a un recurso específico.
  • PUT: Reemplaza y actualiza todas las representaciones del recurso.
  • PATCH: Reemplaza y actualiza datos parcialmente.
  • DELETE: Borra recursos.
  • CONNECT: Establece un túnel al servidor especificado.
  • TRACE: Realiza prueba de búcle de retorno.

• ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST porque es el adecuado para ingresar información cifrada y enviarla al servidor.

No existió error, todo código en rango de 200 significa que la petición fue procesada exitosamente.

• ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No, dado a que el error 404 es derivado de problemas específicamente del usuario dado a que el recurso especificado no existe.

• ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, dado a que los errores 500 son dependientes del servidor en cuestión.

• ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

El deprecated significa que un elemento sigue en probable funcionamiento, pero ya no se le da mantenimiento y no es recomendado para su uso en versiones posteriores puesto a que se actualizó o dejó de ser útil.

Algunos tags despreciados incluyen font (Fuente y tamaño de letra), menu (Especifica una lista de tipo menú), u (subrayar), dir (directorios), etc.

• ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML4 manejaba una variedad extensa de etiquetas para todo el contenido, mezclando la estructura de la página con la presentación de esta, HTML5 puede ahora anexar elementos como SVG y MathML, además de que maneja secciones nuevas como section, article, aside, header, footer. Mientras tanto, HTML4 maneja head, body y las etiquetas originales, asimismo el ahora llamado header antes se escribía con la palabra div y class dentro de los símbolos de identificación, simplificando el proceso ahora en HTML5.

• ¿Qué componentes de estructura y estilo tiene una tabla?

  • caption: Descripcion de la tabla y lo que contiene
  • colgroup: Agrupa columnas por temática
  • col: Aplica atributos comunes
  • thead: Los encabezadps de la tabla
  • tbody: El cuerpo de la tabla
  • tfoot: Pie de tabla.
  • tr: Fila de tabla
  • th: Header en una celda
  • td: Contiene la información de una celda de manera regular.

• ¿Cuáles son los principales controles de una forma HTML5?

INPUT

  • Los campos de texto "input" son los controles de formulario más básicos, permiten al usuario introducir cualquier tipo de datos.
  • Las casillas de verificación se crean estableciendo el atributo type del elemento "input" en el valor checkbox.
  • Un botón de opción se crea estableciendo el atributo type del elemento "input" en el valor radio.


BOTONES

  • submit --> Envía los datos del formulario al servidor.
  • reset --> Restablece todos los controles de formulario a sus valores por defecto.
  • button --> Botones que no tienen efecto automático, pero que se pueden personalizar con código JavaScript.
