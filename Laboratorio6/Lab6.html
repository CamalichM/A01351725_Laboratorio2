¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Porque JavaScript maneja funcionalidades variadas en relación a inputs que garantizan desde la igualdad de las contraseñas hasta la confidencialidad en vista de las mismas, lo que permite un acceso sin errores a las bases de datos de los proyectos y protege la información accedible para el usuario, generando un sitio seguro y controlado.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Deshabilitando JavaScript en el navegador o colocando código que genere un form.submit válido para la url.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque cuando se usa correctamente, la variación de funcionalidades del lenguaje permiten ofrecer tanto funciones variadas, accesibles y efectivas como una interfaz hermanada con CSS que de una experiencia total e inmersiva.
